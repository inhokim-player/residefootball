■ 1) 매일 복붙 점검(프롬프트)
  - “집 PC 로컬만 쓸 때: data/automation_config.json 의 allow_local_token_bypass 가 true 면 127.0.0.1 에서 API 호출 시 브라우저 Bearer 없이도 된다(.env 에 U25_API_TOKEN 있어도). 공개 서버면 false.”
  - “.env 에 API_FOOTBALL_KEY, U25_API_TOKEN, U25_CORS_ORIGINS 가 채워져 있는가?”
  - “data/automation_config.json 의 schedule_times·timezone 이 의도대로인가?”
  - “API 가 떠 있는가? (기본 127.0.0.1:8010 — automation_config 의 api.host/port)”
  - “브라우저는 site/index.html 이고, API 기본 주소가 api.residefootball.com 으로 맞는가?”
  - “CORS 에 실제로 여는 사이트 https:// 도메인이 들어가 있는가?”

■ 2) 홈페이지
  - 메인: site/index.html
  - 루트 index.html → 위 주소로 자동 이동
  - API Bearer: site/index.html 상단 API 버튼(모달)에서만 저장 (별도 admin 페이지 없음)

■ 3) Windows에서 쓰는 .bat (프로젝트 루트)
  go.bat
    - 예전 “한 번에 켜기” 이름. 내부에서 start_u25_api.bat 과 동일(브라우저 + API 8010).

  start_u25_api.bat
    - index.html 연 뒤 start_u25_api.ps1 로 API(8010) 기동. go.bat 과 같은 동작.

  RESIDE_전체자동.bat
    - API + u25_auto_scheduler(설정의 schedule_times·Asia/Seoul) + live_feed 를 Full 로 동시에 띄움.
    - 한국시간 10시 등 자동 수집은 automation_config.json + 스케줄러 창이 켜져 있을 때 동작.

  register_daily_10am_task.bat (+ scripts/register_daily_pipeline_10am.ps1)
    - Windows “작업 스케줄러”에 매일 10:00(로컬 시계) 로 run_u25_pipeline_10am_once.bat 등록.
    - Windows 시간대를 서울로 두면 10:00 = 한국시간 10시. 관리자 권한으로 실행.
    - RESIDE_전체자동(Full)으로 u25_auto_scheduler 가 이미 같은 시각에 돌면 중복 실행될 수 있음 → 둘 중 하나만 쓰거나 시각을 어긋나게 할 것.

  enable_daily_10am_task.bat / disable_daily_10am_task.bat
    - 등록된 U25_RESIDE_DailyPipeline_10am 작업을 수동으로 켜기/끄기.
    - Full 상시 운영(RESIDE_전체자동, RUN_U25_AT_LOGON) 중에는 disable 권장.

  run_u25_pipeline_10am_once.bat
    - 스케줄러 전용: u25_pipeline.py --once (pause 없음). API-Football 수집·DB 반영 한 사이클.

  RUN_U25_AT_LOGON.cmd
    - 로그온 시 start_server_stack.bat Full 호출. 작업 스케줄러 “로그온 시 실행”에 등록 가능.

  start_server_stack.bat  [Full | Api | Scheduler | LiveFeed]  [PublicBind]
    - scripts/server_stack.ps1. Full 이면 API+스케줄+라이브 등 통합.

  start_u25_system.bat
    - api_watchdog + u25_auto_scheduler(08/18/23 인자) + live_feed 를 각각 cmd 창.

  start_u25_full_auto.bat
    - server_stack Full 후 run_automation_stack.py

  start_u25_automation.bat
    - server_stack Scheduler 후 pip + u25_auto_scheduler (08/18/23 고정 인자).

  start_global_u25_system.bat
    - global_u25_harvest → u25_pipeline --once --mode global → uvicorn u25_api

  run_u25_update_once.bat
    - u25_pipeline.py --once (py 없으면 python). 수동 실행 시 창 유지용 pause 포함.

  start_live_feed.bat / start_live_feed_daemon.bat
    - live_feed.py

  predeploy_check.bat / daily_ops_check.bat
    - .env·설정 점검, 작업 스케줄러 점검(daily)

■ 4) 시각·설정 한곳
  - data/automation_config.json 의 schedule_times, timezone (예: 10:00, Asia/Seoul)
  - bat 파일 안에 박힌 스케줄 시각이 있으면 json 과 맞추는 것을 권장.

■ 4b) API-Football 선수 키·국적·나이·키(정확도)
  - 수집: global_u25_harvest.py → DB players_u25 (player_id 숫자 키, 이름, country, age, height_cm 등)
  - 반영: u25_pipeline.py --once 가 하베스트+랭킹까지 한 번에 처리.
  - 화면: site/index.html 이 u25_api 의 /players 등으로 DB 값을 표시.

■ 4c) PC 꺼져도 “사이트를 계속” 보려면
  - 브라우저 HTML 만 원하면: 어디에든 정적 호스팅 가능(지금은 GitHub/Vercel 안 씀).
  - 선수 데이터까지: api.residefootball.com 처럼 **항상 켜 둔 서버**에서 u25_api + DB 가 돌아가야 함. 집 PC 전원만으로는 불가.
